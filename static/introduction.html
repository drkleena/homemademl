Welcome!

Welcome to Homemade Machine Learning! Here you will find tutorials, explanations and explorations into machine learning and data science topics. In this sea of information lies valuable insight and this is where we step in. Data scientists and machine learning engineers are tasked with the mission to gain a peek into the patterns that underly the mountains of data in our lap. Ultimately we want to use this data and the underlying patterns to solve cool problems. Want to tell which images of tumors are canerous? Lets train a classifier on the data! All solvable with today's state of the art methods and most importantly all able to be done on your computer at home.

Instances

An instance is an elementary unit of our data. An instance is a single row in any table or data collection which is made of a set of attribute values. Lets use some data to demonstrate what we're talking about. 

# instance demonstration
%cd Chapter-1/Introduction
import pandas as pd
table = pd.read_csv("./toy.csv")
def display_first_instance(table):
    return table[0:1]
display_first_instance(table)

An instance is an independant collection of attribute values, here our first instance is made of sunny, hot, high, False and no. These are sometimes refered to as 'exemplars'.

Attributes

As you have seen before, an attribute represents an aspect to describe an instance. These are realised in terms of their attribute values. Here our table is made up of 4 attributes outlook, temperature, humidity and windy. The last one 'play' is special, more on this soon. Attributes can also be refered to as 'features'.

Concepts

These are the things we are trying to learn and predict from our data. In the above instance, the last attribute 'Play' is what we are trying to predict, this is called a label or class. In this case we have instances paired with a class label, this is an example of a supervised learning problem, where we have instances and their correct identification. When we are trying to predict the discrete class of an instance, we are trying to classify the instance.

Data Types

Nominal

Nominal values are distinct labels or names. The above table contained attributes that had only distinct labels making them nominal. There is no inherent ordering or scale for these values. Things that are nominal: Gender and age (even though it is numeric, there exists an upper bound - no one lives to 1000, so we can safely assume a fixed number of ages)

Ordinal

Here we have either an implicit or explicit ordering. In the example above our attribute 'Temperature' had the values either hot, mild or cool. From our inherent understanding of temperature we can tell that hot is hotter than mild that is in turn hotter than cool. We can say that hot > mild > cool. Here addition and subtraction along with the other mathematical operations don't make sense. In some cases Ordinal and Nominal attribute values can be confused, sometimes it isn't immidietly clear.

Continuous

Continuous values represent real numbers with an explicit upper bound. Measurements such as distance fall into this category. All of our math operations here are allowed.
